### Integration Gaps (Non-Blocking)

**Gap 1: Spawn gate comms feed events absent (TREE-03)**
- **Affected files:** `workspace_live.ex` lines 1205–1238
- **Detail:** `agent.spawn.gate.requested` and `agent.spawn.gate.resolved` handlers update agent card status but produce no `stream_insert` to the comms feed. Approval gates (Phase 6) do emit comms events, creating an asymmetry.
- **Decision:** Explicitly deferred in Phase 9 Plan 03 SUMMARY — "spawn gate signal handlers do not emit comms events (unlike approval gate) — plan did not require them."
- **Impact:** Operator cannot see spawn gate open/close in comms feed. Card UI correctly shows violet gate panel.

**Gap 2: Awaiting synthesis produces no comms event (LEAD-01)**
- **Affected files:** `agent.ex` lines 2320–2338
- **Detail:** `enter_awaiting_synthesis` and `exit_awaiting_synthesis` cast handlers update agent status and broadcast via `set_status_and_broadcast` but no `stream_insert` fires in workspace_live for comms feed visibility.
- **Impact:** Operator cannot see in the comms feed when leader enters/exits synthesis. Agent card correctly shows indigo pulsing dot and "Awaiting synthesis" label.
