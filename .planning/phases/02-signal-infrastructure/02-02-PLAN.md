```elixir
defp signal_for_workspace?(sig, socket) do
  signal_team_id =
    get_in(sig.data, [:team_id]) ||
      get_in(sig, [Access.key(:extensions, %{}), "loomkin", "team_id"])
  subscribed_teams = socket.assigns[:subscribed_teams] || MapSet.new()
  signal_team_id == nil or MapSet.member?(subscribed_teams, signal_team_id)
end
```

  <action>
    Create `lib/loomkin/teams/team_broadcaster.ex`:

    Module: `Loomkin.Teams.TeamBroadcaster` using GenServer.

    State struct fields (per user decision on Claude's discretion):
    - team_ids: MapSet of subscribed team IDs
    - flush_ref: current Process.send_after timer ref (nil when idle)
    - subscription_ids: list of Jido Signal Bus subscription IDs for cleanup
    - subscribers: %{pid => monitor_ref}
    - buffer: %{streaming: [], tools: [], activity: [], status: []} grouped by signal category

    Constants: @flush_interval_ms 50

    Public API:
    - start_link(opts) -- opts[:team_ids] for initial team set
    - subscribe(broadcaster, pid) -- GenServer.call, monitors pid, idempotent
    - add_team(broadcaster, team_id) -- GenServer.call, adds to team_ids MapSet

    Init:
    - Build initial team_ids MapSet from opts
    - Subscribe to all global bus paths via Topics.global_bus_paths() using Signals.subscribe/1
    - Store all subscription_ids from return values

    Signal classification (classify_signal/1):
    - Critical types (instant forward): "team.permission.request", "team.ask_user.question", "team.ask_user.answered", "agent.error", "agent.escalation", "team.dissolved"
    - Batchable categories: streaming (agent.stream.*), tools (agent.tool.*), status (agent.status, agent.role.changed, agent.queue.updated, agent.usage), activity (everything else)

    handle_info({:signal, %Jido.Signal{}} = sig):
    - Extract team_id from sig.data[:team_id] or sig.extensions["loomkin"]["team_id"]
    - If signal_team_id is nil or in state.team_ids:
      - If critical: broadcast_immediate to all subscribers as {:team_broadcast, %{critical: [sig]}}
      - If batchable: classify, buffer, ensure_flush_timer (only set if flush_ref is nil)

    handle_info(:flush):
    - Emit :telemetry.execute for batch size and queue depth
    - Broadcast buffer to all subscribers as {:team_broadcast, buffer}
    - Reset buffer to empty, set flush_ref to nil

    handle_info({:DOWN, _ref, :process, pid, _reason}):
    - Remove pid from subscribers map

    terminate/2:
    - Cancel flush timer if pending
    - Unsubscribe all subscription_ids via Signals.unsubscribe/1

    Delivery: use send/2 (direct process messages) per user decision.

    IMPORTANT anti-patterns to avoid:
    - Always Process.cancel_timer/1 old flush_ref before setting new one (though we only set when nil)
    - Store EVERY subscription_id from Signals.subscribe/1 return
    - Do NOT call GenServer.call inside handle_info (non-blocking only)

    - Test subscriber cleanup: spawn a process, subscribe it, kill it, verify no crash on next flush
