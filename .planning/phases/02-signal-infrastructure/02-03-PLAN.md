    Add TWO new handle_info clauses before the existing signal handlers:

    ```elixir
    # Critical signals from TeamBroadcaster -- process immediately
    def handle_info({:team_broadcast, %{critical: signals}}, socket) do
      socket = Enum.reduce(signals, socket, &dispatch_signal/2)
      {:noreply, socket}
    end

    # Batched signals from TeamBroadcaster -- process each category
    def handle_info({:team_broadcast, batch}, socket) when is_map(batch) do
      socket =
        batch
        |> Enum.flat_map(fn {_category, signals} -> signals end)
        |> Enum.reduce(socket, &dispatch_signal/2)
      {:noreply, socket}
    end
    ```

    Create a `dispatch_signal/2` helper that takes a single %Jido.Signal{} and socket, and dispatches it to the existing handle_info logic. The existing handle_info clauses match on `{:signal, %Jido.Signal{} = sig}` -- the dispatch_signal helper should call the same processing logic. The simplest approach: have dispatch_signal send the signal to self as the existing format:
    ```elixir
    defp dispatch_signal(signal, socket) do
      # Reuse existing signal handling by matching the same patterns
      # This avoids rewriting 20+ handle_info clauses
      send(self(), {:signal, signal})
      socket
    end
    ```

    WAIT -- that would re-enqueue messages, defeating batching. Better approach: extract the signal processing logic from each handle_info into a defp, and call those from dispatch_signal. BUT that's a massive refactor of 20+ clauses.

    SIMPLEST correct approach: Keep existing handle_info({:signal, ...}) clauses as-is. The batch handler unpacks and re-sends as {:signal, sig} to self(). This IS safe because:
    1. The batch already reduced N raw signals to 1 batch message (the win)
    2. Re-dispatching within the batch is fine -- it's the same process, synchronous
    3. The existing handle_info clauses are already tested and working

    Actually, send/2 to self() inside handle_info will queue messages. Use a fold instead. The cleanest approach:

    ```elixir
    def handle_info({:team_broadcast, %{critical: signals}}, socket) do
      socket = Enum.reduce(signals, socket, fn sig, sock ->
        {:noreply, sock} = handle_signal(sig, sock)
        sock
      end)
      {:noreply, socket}
    end

    def handle_info({:team_broadcast, batch}, socket) when is_map(batch) do
      all_signals = Enum.flat_map(batch, fn {_cat, sigs} -> sigs end)
      socket = Enum.reduce(all_signals, socket, fn sig, sock ->
        {:noreply, sock} = handle_signal(sig, sock)
        sock
      end)
      {:noreply, socket}
    end
    ```

    Where handle_signal/2 is a new defp that contains the dispatch logic from the existing handle_info({:signal, ...}) clauses. HOWEVER, that requires extracting ~20 clauses.

    PRAGMATIC approach (recommended): Use send(self(), {:signal, sig}) for now. The batch already reduced message count by 10-50x (the primary goal). The re-dispatch within a batch is a constant factor, not multiplicative. This can be optimized later. The existing handle_info clauses continue to work unchanged, and the signal_for_workspace? filter is removed since TeamBroadcaster pre-filters.

    ```elixir
    def handle_info({:team_broadcast, %{critical: signals}}, socket) do
      for sig <- signals, do: send(self(), {:signal, sig})
      {:noreply, socket}
    end

    def handle_info({:team_broadcast, batch}, socket) when is_map(batch) do
      for {_category, signals} <- batch, sig <- signals do
        send(self(), {:signal, sig})
      end
      {:noreply, socket}
    end
    ```

    **Step 7: Remove signal_for_workspace? calls from existing handle_info clauses**
    The existing handle_info({:signal, sig}, socket) clauses check signal_for_workspace?(sig, socket). Since TeamBroadcaster now pre-filters by team_id, remove these guards from ALL existing signal handle_info clauses. Search for `signal_for_workspace?` and remove the conditional wrapping. This is critical -- without this, signals forwarded by the broadcaster would still be filtered (and they've already been filtered).

    **Step 8: Update terminate/2 (line 100-106)**
    Add broadcaster cleanup:

    ```elixir
    def terminate(_reason, socket) do
      if broadcaster = socket.assigns[:broadcaster] do
        DynamicSupervisor.terminate_child(Loomkin.Teams.BroadcasterSupervisor, broadcaster)
      end

        Loomkin.Permissions.TrustPolicy.cleanup(session_id)
      end

      :ok
    end
    ```
